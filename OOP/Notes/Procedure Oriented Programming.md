Conventional programming using high-level languages such as `FORTRAN` and `C` is commonly known as procedure oriented programming (`POP`). Here the problem is viewed as a sequence of things to be done such as reading, writing and printing.

We organise this instructions into groups known as functions. Here we give little attention to the data that is being used by various functions. 

In a multi function program, there are several global variables which are used by multiple functions. Each function also has its own local variables. Global data is more vulnerable to change by a function. In case we revise the external data structure then we also need to modify all the functions that used to use this data. This could lead to bugs.
